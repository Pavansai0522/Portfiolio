# Using Postman with Portfolio API

## Setup

1. **Start the backend server:**
   ```bash
   npm run start:backend
   ```
   The server will run on `http://localhost:3001`

2. **Import the collection:**
   - Open Postman
   - Click "Import" button
   - Select `postman_collection.json` file
   - The collection will be imported with all endpoints

## API Endpoints

### Base URL
```
http://localhost:3001/api
```

### 1. Health Check
**GET** `/api/health`

Check if the API is running.

**Response:**
```json
{
  "status": "ok",
  "message": "Portfolio API is running"
}
```

---

### 2. Get Portfolio Data
**GET** `/api/portfolio`

Retrieve all portfolio data.

**Response:**
```json
{
  "name": "John Doe",
  "title": "Full Stack Developer",
  "bio": "...",
  "email": "john@example.com",
  "phone": "+1 234 567 8900",
  "location": "San Francisco, CA",
  "profileImage": null,
  "skills": ["JavaScript", "TypeScript"],
  "projects": [...],
  "socialLinks": {...}
}
```

---

### 3. Update Portfolio Data
**PUT** `/api/portfolio`

Update portfolio information. You can send partial data - only the fields you want to update.

**Request Body:**
```json
{
  "name": "John Doe",
  "title": "Full Stack Developer",
  "bio": "Updated bio text",
  "email": "john@example.com",
  "phone": "+1 234 567 8900",
  "location": "San Francisco, CA",
  "skills": ["JavaScript", "TypeScript", "Angular"]
}
```

**Response:** Returns the updated portfolio data

---

### 4. Add Project
**POST** `/api/portfolio/projects`

Add a new project to the portfolio.

**Request Body:**
```json
{
  "title": "E-Commerce Platform",
  "description": "A full-stack e-commerce application",
  "technologies": ["Angular", "Node.js", "MongoDB"],
  "image": null,
  "link": "https://example.com/project"
}
```

**Response:**
```json
{
  "id": 1234567890,
  "title": "E-Commerce Platform",
  "description": "A full-stack e-commerce application",
  "technologies": ["Angular", "Node.js", "MongoDB"],
  "image": null,
  "link": "https://example.com/project"
}
```

**Note:** The `id` is automatically generated by the server.

---

### 5. Update Project
**PUT** `/api/portfolio/projects/:id`

Update an existing project. Replace `:id` with the actual project ID.

**URL Example:** `http://localhost:3001/api/portfolio/projects/1234567890`

**Request Body (partial update):**
```json
{
  "title": "Updated Project Title",
  "description": "Updated description"
}
```

**Response:** Returns the updated project

---

### 6. Delete Project
**DELETE** `/api/portfolio/projects/:id`

Delete a project by ID.

**URL Example:** `http://localhost:3001/api/portfolio/projects/1234567890`

**Response:**
```json
{
  "message": "Project deleted successfully"
}
```

---

## Testing Tips

1. **Start with Health Check** - Verify the server is running
2. **Get Portfolio Data** - See the current state
3. **Update Portfolio** - Modify your information
4. **Add Projects** - Create new projects
5. **Update/Delete Projects** - Manage existing projects

## Headers

For PUT and POST requests, make sure to set:
```
Content-Type: application/json
```

## Common Issues

- **Connection Refused**: Make sure the backend server is running (`npm run start:backend`)
- **404 Not Found**: Check the URL path and ensure you're using the correct endpoint
- **400 Bad Request**: Verify your JSON body is valid and matches the expected structure
